# Database Schema Reference
## Smart Classroom Assistant — All Collections & Vector Store

> **Database Engine (Primary):** MongoDB (NoSQL, Document-Oriented) — Database Name: `smart_classroom`  
> **Database Engine (Vector):** Qdrant (Embedded Local Mode) — Collection: `classroom_knowledge`  
> **Total Collections:** 13 MongoDB Collections + 1 Qdrant Collection

## MongoDB Collections

---

### 1. Collection: `users`

**Purpose:** Stores authentication credentials, profile information, and enrollment metadata for every user in the system — teachers, students, and staff. This is the primary identity store queried on every authenticated API request.

**Indexes:**
- `email` — Unique index (ascending)

| Field Name | Data Type | Constraints |
|:---|:---|:---|
| `_id` | ObjectId | Primary Key; auto-generated by MongoDB on document creation |
| `email` | String | **Required**; must be globally unique; stored in lowercase; indexed |
| `password_hash` | String | **Required**; bcrypt hash of the user's password (cost factor 12); never stored in plaintext |
| `full_name` | String | **Required**; user's display name; max 150 characters |
| `role` | String | **Required**; enforced enum: `"teacher"` \| `"student"` \| `"staff"` |
| `department` | String | Optional; e.g., `"Computer Science"`, `"Electronics"` |
| `year` | String | Optional; applicable to students only; e.g., `"1"`, `"2"`, `"3"`, `"4"` |
| `section` | String | Optional; applicable to students only; e.g., `"A"`, `"B"` |
| `enrolled_subjects` | Array[String] | Default: `[]`; list of `subject_id` strings the student is enrolled in; used for access control filtering |
| `enrolled_sessions` | Array[String] | Default: `[]`; list of `session_id` strings the student has been associated with |
| `profile_picture_url` | String | Optional; relative server path or external URL to the user's profile image |
| `created_at` | DateTime (UTC) | **Required**; automatically set at the time of document creation |
| `last_login` | DateTime (UTC) | Optional; updated on each successful authentication event |
| `is_active` | Boolean | Default: `true`; set to `false` to soft-disable a user account without deletion |

### 2. Collection: `subjects`

**Purpose:** Defines the taxonomy of academic courses offered within the institution. Every session, material, quiz, and assignment is scoped to a subject. The subject record is the foundational enrollment unit for students.

**Indexes:**
- Compound unique index on `(subject_name, department, year, section, academic_year)`
- `subject_id` — Unique index (ascending)

| Field Name | Data Type | Constraints |
|:---|:---|:---|
| `_id` | ObjectId | Primary Key; auto-generated by MongoDB |
| `subject_id` | String (UUID) | **Required**; application-generated UUID; globally unique; used as the primary foreign key across all related collections |
| `subject_name` | String | **Required**; full name of the course; e.g., `"Data Structures and Algorithms"` |
| `subject_code` | String | Optional; institutional code; e.g., `"CS401"`, `"EC301"` |
| `department` | String | **Required**; e.g., `"Computer Science"`, `"Electrical Engineering"` |
| `year` | String | **Required**; academic year of study; e.g., `"1"`, `"2"`, `"3"`, `"4"` |
| `section` | String | **Required**; class division; e.g., `"A"`, `"B"`, `"C"` |
| `semester` | String | Optional; e.g., `"Odd"`, `"Even"`, `"1"`, `"2"` |
| `academic_year` | String | **Required**; e.g., `"2025-2026"` |
| `faculty_name` | String | **Required**; full name of the assigned instructor |
| `faculty_id` | String | **Required**; corresponds to the faculty user's `email` or user ID |
| `schedule` | Array[Object] | Optional; weekly timetable slots: `{day: String, period: String, start_time: String, end_time: String, room: String}` |
| `created_at` | DateTime (UTC) | Auto-set on document creation |
| `is_active` | Boolean | Default: `true`; set to `false` for archived or discontinued subjects |

### 3. Collection: `sessions`

**Purpose:** The central and most data-rich collection in the system. Each document represents a single live lecture session. It accumulates real-time transcript segments during the session, and post-session processing adds AI-generated summaries, concept lists, audio summaries, and PDF report references.

**Indexes:**
- `session_id` — Unique index (ascending)
- `started_at` — Descending index (for time-sorted queries)
- `status` — Ascending index (for filtering active sessions)
- `subject_id` — Ascending index (for subject-scoped session retrieval)

| Field Name | Data Type | Constraints |
|:---|:---|:---|
| `_id` | ObjectId | Primary Key; auto-generated by MongoDB |
| `session_id` | String | **Required**; application-generated ID format: `{subject_id}-{YYYYMMDD_HHMMSS}`; globally unique |
| `subject_id` | String | **Required**; foreign key → `subjects.subject_id` |
| `subject_name` | String | **Required**; denormalized from `subjects` for efficient display without joins |
| `teacher_email` | String | **Required**; email of the teacher who initiated the session; foreign key → `users.email` |
| `started_at` | DateTime (UTC) | **Required**; timestamp when the teacher clicked "Start Session" |
| `ended_at` | DateTime (UTC) | Set when the teacher stops the session; `null` while session is active |
| `status` | String | **Required**; enforced enum: `"active"` \| `"processing"` \| `"finished"`; default: `"active"` |
| `transcripts` | Array[Object] | Default: `[]`; real-time transcript segments: `{text: String, timestamp: DateTime}` — appended via `$push` for each Whisper output segment |
| `transcript_chunks` | Array[String] | Legacy format; plain string array for older sessions; normalized with `transcripts` at retrieval |
| `teaching_materials` | Array[Object] | Default: `[]`; materials shared during the session: `{id: String (UUID), title: String, url: String, type: String ("image"\|"pdf")}` |
| `current_context` | Object | Tracks the material currently displayed to students: `{id: String, type: String, url: String}` or `{type: "none"}` |
| `summary` | String | Post-session; LLM-generated executive summary of the lecture; set by the finalization pipeline |
| `concepts` | Array[String] | Post-session; key academic concepts extracted by the LLM; e.g., `["Binary Search", "Complexity Analysis"]` |
| `questions` | Array[Object] | Post-session; practice questions generated from the transcript by the LLM |
| `session_type` | String | Post-session; LLM-classified session category; e.g., `"Lecture"`, `"Lab"`, `"Workshop"`, `"Tutorial"` |
| `taxonomy` | Object | Post-session; structured concept taxonomy from LLM analysis; e.g., `{topic: String, subtopics: Array}` |
| `audio_summary_url` | String | Post-session; relative server path to the generated MP3 podcast audio file; e.g., `"/storage/audio/session_xxx.mp3"` |
| `pdf_report_url` | String | Optional; relative server path to the generated FPDF2 PDF report |
| `analyzed_at` | DateTime (UTC) | Timestamp when the post-session AI analysis pipeline completed |
| `is_indexed` | Boolean | Default: `false`; set to `true` once transcript chunks are embedded and stored in Qdrant |

### 4. Collection: `quizzes`

**Purpose:** Stores all assessment documents — both session-scoped pop quizzes generated during live class and comprehensive subject-level assessments drawn from multiple sessions via the RAG pipeline. The teacher can review, edit, save, and publish them. Students submit answers against these documents.

| Field Name | Data Type | Constraints |
|:---|:---|:---|
| `_id` | ObjectId | Primary Key; auto-generated by MongoDB |
| `quiz_id` | String (UUID) | **Required**; application-generated UUID; globally unique; used as submission foreign key |
| `title` | String | **Required**; descriptive title; e.g., `"Assessment: Data Structures"`, `"Comprehensive Assessment: Unit 2"` |
| `subject_id` | String | **Required** (for comprehensive); foreign key → `subjects.subject_id` |
| `session_id` | String | For session-scoped quizzes; foreign key → `sessions.session_id` |
| `session_ids` | Array[String] | For multi-session comprehensive quizzes; array of `sessions.session_id` values used as RAG source |
| `is_comprehensive` | Boolean | Default: `false`; `true` for multi-session subject-level assessments; `false` for single-session pop quizzes |
| `questions` | Array[Object] | **Required**; ordered array of question sub-documents (see Question Sub-Schema below) |
| `total_max_points` | Integer | **Required**; sum of `max_points` across all questions; auto-computed on save |
| `created_by` | String | **Required**; email of the teacher who created the quiz; foreign key → `users.email` |
| `created_at` | DateTime (UTC) | Auto-set on document creation |

**Question Sub-Document Schema (within `questions` array):**

| Field Name | Data Type | Constraints |
|:---|:---|:---|
| `id` | String | **Required**; unique within the quiz; e.g., `"q1"`, `"q2"` |
| `type` | String | **Required**; enforced enum: `"mcq"` \| `"short_answer"` |
| `question` | String | **Required**; the question text displayed to the student |
| `options` | Array[String] | **Required for `type = "mcq"` only**; exactly 4 option strings |
| `correct_index` | Integer | **Required for `type = "mcq"` only**; zero-based index (0–3) of the correct option |
| `max_points` | Integer | **Required**; scoring weight — default `2` for MCQ, `10` for short answer |
| `model_answer` | String | **Required for `type = "short_answer"` only**; LLM-generated reference answer used as grading baseline |
| `source_evidence` | String | Optional; verbatim excerpt from the lecture transcript that grounds the question |

### 5. Collection: `quiz_submissions`

**Purpose:** Records each student's attempt on a quiz, storing their raw answers alongside the AI-computed grading results on a per-question basis. Enforced single-submission policy prevents duplicate attempts.

| Field Name | Data Type | Constraints |
|:---|:---|:---|
| `_id` | ObjectId | Primary Key; auto-generated by MongoDB |
| `submission_id` | String (UUID) | Application-generated UUID; globally unique |
| `quiz_id` | String (UUID) | **Required**; foreign key → `quizzes.quiz_id` |
| `quiz_title` | String | Denormalized from `quizzes.title` for display efficiency |
| `subject_id` | String | **Required**; denormalized from `quizzes.subject_id` for efficient subject-scoped querying |
| `session_id` | String | Optional; denormalized for legacy session-scoped queries |
| `student_email` | String | **Required**; email of the submitting student; foreign key → `users.email`; composite uniqueness with `quiz_id` enforced in application logic |
| `student_name` | String | Denormalized from `users.full_name` for display efficiency |
| `answers` | Object | Map of `question_id → student_answer` (String or Integer for MCQ index) — raw ungraded submission |
| `results` | Array[Object] | Per-question grading output (see Result Sub-Schema below) |
| `total_score` | Float | **Required**; computed sum of all per-question scores |
| `max_score` | Float | **Required**; `quizzes.total_max_points` at the time of submission |
| `average_score` | Float | **Required**; stored as a percentage value: `(total_score / max_score) × 100` |
| `submitted_at` | DateTime (UTC) | Auto-set when the submission is persisted |

**Result Sub-Document Schema (within `results` array):**

| Field Name | Data Type | Constraints |
|:---|:---|:---|
| `question_id` | String | References `quizzes.questions[n].id` |
| `score` | Float | Awarded score for this question; weighted from raw AI score (0–10) to `max_points` scale |
| `max_points` | Integer | Maximum points possible for this question |
| `is_correct` | Boolean | For MCQ only; `true` if selected index matches `correct_index` |
| `feedback` | String | For MCQ: static correct/incorrect message with answer reveal; for short answer: LLM-generated personalized feedback |
| `teacher_quote` | String | Short answer only; verbatim excerpt from lecture context cited by the LLM to validate or correct the student's answer |
| `evidence` | String | MCQ only; the `source_evidence` field from the original question |

## Qdrant Vector Database

---

### 14. Qdrant Collection: `classroom_knowledge`

**Purpose:** The semantic knowledge base for the Retrieval-Augmented Generation (RAG) pipeline. Every finished session's transcript is chunked into 500-character overlapping segments and every uploaded PDF is parsed page-by-page. Each text chunk is then converted into a 384-dimensional dense vector embedding using the `all-MiniLM-L6-v2` Sentence Transformer model and stored here. At query time, the system performs approximate nearest-neighbour (ANN) cosine similarity search, filtered by `session_id`, to retrieve the most contextually relevant chunks for quiz generation, AI Q&A, and answer grading.

**Storage Location:** `backend/storage/qdrant_db/` (embedded local mode)  
**Vector Dimensions:** 384  
**Distance Metric:** Cosine Similarity  
**Payload Index:** `session_id` (KEYWORD type) — enables fast, filtered retrieval scoped to specific sessions

| Field Name | Data Type | Constraints |
|:---|:---|:---|
| `id` | String (UUID) | **Primary Key**; auto-generated UUID assigned to each vector point at insertion |
| `vector` | Float[384] | **Required**; 384-dimensional dense embedding produced by `all-MiniLM-L6-v2`; represents the semantic meaning of the associated text chunk; used for cosine similarity search |
| `payload.text` | String | **Required**; the raw text chunk this vector represents; returned in search results as the grounding context for LLM prompts |
| `payload.session_id` | String | **Required**; the `session_id` of the session this chunk belongs to; **keyword-indexed** for fast filtered retrieval; all points for a session are deleted and re-inserted on re-indexing to maintain idempotency |
| `payload.type` | String | **Required**; source content type: `"transcript"` (from live session transcription) or `"material"` (from uploaded PDF page extraction) |
| `payload.chunk_index` | Integer | **Required for `type = "transcript"`**; zero-based sequential position of this chunk within the reconstructed full session transcript; preserves reading order |
| `payload.material_id` | String | **Required for `type = "material"` only**; corresponds to the `id` field within `sessions.teaching_materials` array — identifies the source material file |
| `payload.page` | Integer | **Required for `type = "material"` only**; 1-based page number of the source PDF from which this chunk was extracted |
| `payload.title` | String | **Required for `type = "material"` only**; title of the source teaching material document for display in RAG results |

**Chunking Parameters:**

| Parameter | Value | Description |
|:---|:---|:---|
| `chunk_size` | 500 characters | Maximum character length of each text chunk |
| `overlap` | 100 characters | Number of characters shared between consecutive chunks to preserve semantic continuity at boundaries |
| **Effective stride** | 400 characters | `chunk_size - overlap`; distance between start positions of consecutive chunks |
